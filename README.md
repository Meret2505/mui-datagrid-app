# 📚 Книжный каталог — MUI DataGrid

Учебное React-приложение, демонстрирующее работу с библиотекой компонентов
**Material UI** и таблицей **DataGrid** (`@mui/x-data-grid`). Данные о книгах
подгружаются из открытого **Open Library API**.

## 🔗 Демо

> Замените на свою ссылку после деплоя:
> **[Открыть приложение](https://your-app.netlify.app)**

## ✨ Возможности

### Основное

- ⚛️ **React 18** + **Material UI 6** + **Vite**.
- 📊 Таблица на компоненте **`DataGrid`** из MUI.
- 🧱 **6 колонок**: обложка (картинка), название и автор (текст), дата издания
  (дата), рейтинг и количество страниц (числа).
- 🔍 Все колонки **сортируются и фильтруются** (плюс быстрый поиск в тулбаре).
- 🎨 У текстовых колонок **разные стили**: название — жирный serif в цвете
  primary, автор — курсив в цвете secondary, дата — моноширинный шрифт.
- 📐 **Автоматическая высота строк** (`getRowHeight: 'auto'`) с ограничением
  `min 100px / max 300px` через CSS.
- 🖱️ **Клик по строке** → модальное окно с подробной информацией о книге.
- 🖼️ **Клик по обложке** → модальное окно с **увеличенным изображением**.
- 🧩 Использованы и **функциональные** компоненты (`RowDetailModal`,
  `BooksTable`, `App`), и **классовый** компонент (`ImageModal`).

### Бонусы ✅

- 💾 **Сохранение состояния таблицы** (сортировка, фильтры, пагинация) в
  `localStorage` — переживает перезагрузку страницы.
- 🌗 **Переключение тёмной / светлой темы** (тоже сохраняется).
- 🌐 Реальные данные из **открытого API** (Open Library).

## 🚀 Запуск локально

```bash
npm install
npm run dev      # http://localhost:5173
```

Сборка production-версии:

```bash
npm run build
npm run preview
```

## 📁 Структура проекта

```
src/

├── theme/
│   └── theme.js                # фабрика light/dark темы MUI
├── hooks/

    ├── RowDetailModal.jsx      # модалка строки (функц. компонент)

```

## 🌍 Деплой

### Netlify

1. Подключите репозиторий на [netlify.com](https://app.netlify.com).
2. Build command: `npm run build`, publish directory: `dist`
   (уже задано в `netlify.toml`).

### GitHub Pages

В репозитории есть workflow `.github/workflows/deploy.yml`. Включите Pages
(Settings → Pages → Source: GitHub Actions), и при пуше в `main` приложение
задеплоится автоматически.
